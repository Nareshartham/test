Abstracting Code with Functions

Pieces of computer programs are referred to as “code”. Recall the code for the area of a square, the sequence of assignment statements used in the calculation of the area, where the length of each side is 3 inches.

side = 3
area = side * side

In this example, rather than using the assignment statement area = 3 * 3, we used the expression side * side on the right hand side. This is a simple form of abstraction: whenever we write an expression that stands for the area of a square, we assume that the length of side has somehow been defined and only need to remember that the area is computed by taking the square of the length of its side. We don’t need to worry about the specific value of side. There is still something unsatisfactory about how we compute area. If we wanted to compute the area of a square of a different size we would need to replicate a similar sequence of steps.



You can see that the only thing that changes in our code is the value we assign to the variable side. Now we will learn a way to turn this sequence of assignments into a more general and reusable piece of Python code called a function.

Below is a function definition in Python that encapsulates instructions to compute the area of a square of any size.



The keyword def indicates that we are defining a function. sq_area is the name that refers to our function and side is a parameter, which stands for the side length which the function uses to compute the area. Our goal in defining this function is to be able to replace the parameter with any actual value for the side length and get the corresponding square area in return. In order to compute the area of a square of size 3 inches, we would invoke the function as sq_area(3), in order to compute the area of a square of size 4 inches we would invoke as sq_area(4), and so on.

The indented part of the function definition is called the function body. In this example, it consists of a return statement, which we will look at more closely later.

Any function definition in Python should conform to the following syntactic form.

def function name (list of formal parameters)
 function body

Note that a function is an abstraction that defines how the computation will be performed. The computation defined by the function body is actually executed when the function is called (used) by replacing the formal parameters with the arguments. Arguments are the actual values that are substituted for the formal parameters in the function body during execution.
