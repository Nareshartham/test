Testing Functions and Did I get this topic?


Testing functions:

Programs are rarely correct when they are first written. In addition to writing the code, you should test your code on multiple inputs, for which you independently know the correct output. For example, to test for sq_area(side), you can call it with different values for side and see that each call indeed returns the square of that value: sq_area(5) should return 25, sq_area(10) should return 100 and so on.

Test the functions given in the previous activity to convince yourself that they give the same output as sq_area on each legal input (positive integer).

Enter the function definitions given in the text above in the left editor pane. Then call them in the shell with different inputs, for which you know what to expect as an answer. For example, when you type a(10) in the shell in the right pane, it should give you 100, and so should sq_area(10).
