Comments

Comments:


We mentioned earlier that choosing informative names for variables and functions is important to enhance readability of code. Another way to enhance readability is to add comments to the code, which are explanatory notes intended for human readers. The Python interpreter ignores all the text that comes after the symbol # to the end of the line. For example, we could add a comment to the definition of sq_area indicating our assumption about the type of the argument and stating what the function is supposed to do. 

def sq_area(side):
# assumes side is a positive integer or float
    return side * side # returns square of side


In this course, whenever the code is self-explanatory or is explained in sufficient detail in the surrounding discussion we wont include the comments. However, we recommend that you make generous use of comments in writing your own code.
