Abstracting Code with Functions


Let us look at the following function definition more closely.

def sq_perimeter(side):
    return 4 * side


When we call this function with an argument, say 20, the function body is executed with 20 substituted for the parameter side. This means the actual statement that is executed when the function is called is return 4 * 20. Since 4 * 20 is 80, the function returns 80.

Body of the function definitionExecuted statement
return 4 * side return 4 * 20


Function bodies may consist of multiple statements that are executed in the order they appear in the code. For example, the following function also computes the perimeter of a square. The only difference it has from sq_perimeter is that it uses an assignment statement as an intermediate step before returning the result. It computes the result, gives it a name (perimeter) and returns the value that is bound to that name. 

def sq_perimeter2(side):
    perimeter = 4 * side
    return perimeter


Note that the function body, which is the sequence of statements that describe the computation to be done when the function is executed, must be indented as above. Indenting all the statements in the function body is a requirement in Python. You can use as many spaces as you want while it is customary to use 4 spaces in the Python community.
